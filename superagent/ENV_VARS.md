# SUPERAGENT — Environment Variables Reference

Every environment variable the system uses. Required variables are marked with ⚠️.

---

## Application

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `APP_ENV` | No | `production` | Runtime environment: `production`, `development`, `testing` |
| `APP_PORT` | No | `8000` | Port the HTTP server listens on. Render sets `PORT` automatically. |
| `APP_HOST` | No | `0.0.0.0` | Bind address |
| `APP_VERSION` | No | `unknown` | Application version string (set by CI/CD at build time) |
| `LOG_LEVEL` | No | `INFO` | Logging verbosity: `DEBUG`, `INFO`, `WARNING`, `ERROR` |
| `SECRET_KEY` | ⚠️ Yes | — | Cryptographic key for signing sessions, JWTs, and internal tokens. Auto-generated by Render if using blueprint. Generate: `python -c "import secrets; print(secrets.token_urlsafe(64))"` |
| `DEBUG` | No | `false` | Enable debug mode. **Never enable in production.** |

---

## Database

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `DATABASE_URL` | No | `sqlite:///data/superagent.db` | Connection string. Supports `sqlite:///` and `postgresql://` |

### SQLite Format
```
DATABASE_URL=sqlite:///data/superagent.db
```

### PostgreSQL Format
```
DATABASE_URL=postgresql://user:password@host:5432/dbname
```

Render's managed Postgres auto-sets this variable if you add a Postgres service.

---

## Redis

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `REDIS_URL` | No | `redis://localhost:6379/0` | Redis connection string. Used for sessions, caching, rate limiting. |

### Upstash (Serverless Redis)
```
REDIS_URL=rediss://default:xxxx@region.upstash.io:6379
```
Note: `rediss://` (double s) for TLS.

### Render Redis
Auto-injected from blueprint — no manual setup needed.

### Local Docker
```
REDIS_URL=redis://redis:6379/0
```

## LLM API

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `OPENROUTER_API_KEY` | ⚠️ Yes | — | OpenRouter API key. Get at [openrouter.ai/keys](https://openrouter.ai/keys). Routes to cheapest/fastest provider. |
| `LLM_MODEL` | No | `qwen/qwen3.5-9b` | Default model for general tasks |
| `LLM_MODEL_COMPLEX` | No | `deepseek/deepseek-v4-pro` | Model for complex reasoning tasks |
| `LLM_MODEL_FAST` | No | `openai/gpt-oss-20b` | Cheapest model for classification/routing |
| `LLM_MAX_TOKENS` | No | `4096` | Max output tokens per request |
| `LLM_TEMPERATURE` | No | `0.7` | Default temperature |

### OpenRouter Model IDs
Common budget models (as of July 2026):
```
qwen/qwen3.5-9b          # $0.17/$0.25 per 1M tokens — budget king
openai/gpt-oss-20b        # $0.05/$0.20 — cheapest OpenAI
google/gemma-4-31b        # $0.39/$0.97 — good value
minimax/minimax-m3        # $0.30/$1.20 — great all-rounder
deepseek/deepseek-v4-pro  # $1.74/$3.48 — best reasoning
```

### Using Together AI Instead
```
OPENROUTER_API_KEY=         # leave empty
TOGETHER_API_KEY=your-key   # set this instead
LLM_BASE_URL=https://api.together.xyz/v1
```

## Telegram

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `TELEGRAM_BOT_TOKEN` | ⚠️ Yes | — | Bot token from [@BotFather](https://t.me/BotFather). Format: `123456789:ABCdef...` |

| `TELEGRAM_WEBHOOK_SECRET` | No | (auto-generated) | Secret token for webhook verification. Generate: `python -c "import secrets; print(secrets.token_hex(32))"` |
| `TELEGRAM_ALLOWED_USERS` | No | (all) | Comma-separated Telegram user IDs. If set, only these users can interact. |

## Monitoring & Observability

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `HELICONE_API_KEY` | No | — | [Helicone](https://helicone.ai) API key for LLM cost tracking. Free: 100K requests/month. |
| `SENTRY_DSN` | No | — | [Sentry](https://sentry.io) DSN for error tracking. Free: 5K events/month. |
| `SENTRY_TRACES_SAMPLE_RATE` | No | `0.1` | Sentry performance tracing sample rate (0.0–1.0) |

### Helicone Setup
Add to OpenRouter requests via header — the app does this automatically if `HELICONE_API_KEY` is set:
```
Helicone-Auth: Bearer ${HELICONE_API_KEY}
```

## Example .env

```bash
# === Required ===
SECRET_KEY=your-random-secret-key-here
OPENROUTER_API_KEY=sk-or-v1-xxxxxxxxxxxx
TELEGRAM_BOT_TOKEN=123456789:ABCdefGHIjklMNOpqrSTUvwxYZ

# === App ===
APP_ENV=production
APP_PORT=8000
LOG_LEVEL=INFO

# === Data ===
DATABASE_URL=sqlite:///data/superagent.db
REDIS_URL=redis://localhost:6379/0
CHROMA_URL=http://localhost:8100

# === LLM ===
LLM_MODEL=qwen/qwen3.5-9b
LLM_MODEL_FAST=openai/gpt-oss-20b

# === Monitoring (optional) ===
HELICONE_API_KEY=your-helicone-key
SENTRY_DSN=https://xxx@sentry.io/xxx

# === Rate Limits ===
RATE_LIMIT_PER_MINUTE=30
RATE_LIMIT_PER_DAY=500
```

## Security Notes

1. **Never commit `.env` to git.** It's in `.gitignore` by default.
2. **Rotate `SECRET_KEY`** if it's ever exposed.
3. **Use `TELEGRAM_ALLOWED_USERS`** in production to restrict access.
4. **Use TLS** for all external connections (Render auto-provides HTTPS).
5. **Redis auth:** Use `rediss://` (TLS) for remote Redis connections.
6. **API keys:** Treat `OPENROUTER_API_KEY` like a password. It has billing access.
